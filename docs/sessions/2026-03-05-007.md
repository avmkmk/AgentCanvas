### Decisions Made

| Decision | Rationale | Alternatives |
|----------|-----------|-------------|
| ExecutionResponse rewritten to match ORM model fields | model_validate() requires exact field name alignment; existing schema had different field names (input_data, step_count, created_at) that don't exist on the ORM | Could use field aliases but that adds complexity and hides the mismatch |
| StepExecution records created during _execute_agents | Provides per-step auditability; model already existed from M2 DB migrations | Could skip step records and only track FlowExecution — deferred feature audit |
| Background task uses _AsyncSessionLocal directly | Request-scoped session is closed after HTTP response; background tasks need their own session | Could create a separate get_background_db helper — DRY applies here |
| memory endpoints return empty agent_memories dict | Full per-agent aggregation is a M4 concern when frontend needs it | Could aggregate all agent keys from MongoDB but no UI consumer exists yet |
| ws_manager is in-process only | MVP constraint; Redis pub/sub added in M4 for horizontal scaling | Redis pub/sub considered but out of scope for M3 |
