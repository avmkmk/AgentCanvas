### Decisions Made

| Decision | Rationale | Alternatives |
|----------|-----------|-------------|
| Test deps in single `requirements.txt` | Single-stage Dockerfile — splitting adds complexity with no current benefit | `requirements-test.txt` deferred to M6 multi-stage build |
| `/// <reference types="vitest" />` triple-slash in vite.config | Gives TypeScript the `test` property type without changing the `defineConfig` import source | Import `defineConfig` from `vitest/config` — changes import source, more disruptive |
| `asyncio_mode = "auto"` global | Eliminates decorator noise on all 62 existing tests; matches project convention | Per-test `@pytest.mark.asyncio` — verbose, error-prone |
