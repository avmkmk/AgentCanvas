### Decisions Made

| Decision | Rationale | Alternatives |
|----------|-----------|-------------|
| Two git worktrees for backend and frontend | Enables independent PRs per track; keeps branch histories clean | Single branch for everything — harder to review per domain |
| Reuse shared `_AsyncSessionLocal` in health.py | Avoids second unmanaged engine; ensures consistent pool config | Separate lightweight engine (rejected — resource leak risk) |
| `hmac.compare_digest` at module level import | Coding Standard 7 — import at top; no per-request dict lookup | Inner import (was rejected, fixed in review) |
| `allow_headers=["Content-Type","X-API-Key"]` | Explicit CORS headers match SECURITY.md recommendation | `["*"]` — too permissive, fixed in review |
| Polling for HITL/execution in M1 (not WebSocket) | WebSocket infra (BC-08, FE-11) is M3; polling sufficient for MVP demo | WebSocket — deferred to M3 |
| QA agent wrote tests for WRITE NOW modules | 5 modules have security validators that must be verified (injection guards, timing-safe compare) | Defer all tests to M6 — rejected because security validators are untested risk |
| `VITE_API_KEY` security note in apiClient.ts | Known M1 limitation; documented inline + references issue #29 for token auth in M2 | Remove API key from FE entirely — deferred to BFF/token auth work |
| `flow_config` defaults to `{"nodes": [], "edges": []}` should be enforced by backend router | Noted in integration report as a follow-up for BA-02 (Flow CRUD) | Empty dict `{}` default causes frontend narrowing issue at render |
